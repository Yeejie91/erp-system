# ERP系统使用指南

## 系统概述

这是一个专为中小企业设计的ERP管理系统，支持商品管理、库存管理、进出货管理、活动管理、财务管理等核心功能。

## 快速开始

### 1. 启动系统

```bash
cd erp-system
npm install  # 首次使用需要安装依赖
npm run dev  # 启动开发服务器
```

系统将在浏览器中自动打开 `http://localhost:3001`

### 2. 首次使用建议流程

**第一步：添加商品**
1. 进入"商品管理"模块
2. 点击"添加商品"按钮
3. 填写商品信息（名称、SKU、分类、单位等）
4. 设置库存预警值（建议设置为安全库存量）
5. 保存商品

**第二步：添加客户和供应商**
1. 进入"客户管理"添加主要客户信息
2. 进入"供应商管理"添加供应商信息
3. 这些信息会在后续的订单和活动中使用

**第三步：初始化库存**
1. 进入"进货管理"
2. 为已有商品录入初始库存
3. 系统会自动更新库存数量

## 核心功能详解

### 📦 商品管理

**功能说明：**
- 管理所有商品的基础信息
- 支持SKU编号、分类、单位管理
- 设置成本价和销售价
- 库存预警功能

**操作步骤：**
1. 点击"添加商品"
2. 填写必填项：名称、SKU、分类、单位、预警值
3. 选填：成本价、售价、供应商、描述
4. 保存后可随时编辑或删除

**注意事项：**
- SKU编号建议使用有规律的编码方式
- 库存预警值建议设置为正常销售量的2-3倍
- 当库存低于预警值时，系统会在多处显示预警

### 📊 库存管理

**功能说明：**
- 实时查看所有商品库存状态
- 库存价值统计
- 低库存商品预警
- 查看库存历史记录

**关键指标：**
- **当前库存**：实时库存数量
- **库存状态**：通过进度条和颜色显示库存健康度
- **库存价值**：当前库存 × 成本价
- **库存记录**：所有进出库的历史记录

### ⬆️ 进货管理

**功能说明：**
- 记录商品采购入库
- 支持批量进货
- 自动更新库存
- 生成进货记录

**操作流程：**
1. 点击"新增进货"
2. 选择商品并输入数量
3. 可添加多个商品到进货清单
4. 填写操作人和备注
5. 确认进货后自动更新库存

**应用场景：**
- 从供应商采购商品
- 初始化商品库存
- 退货入库

### ⬇️ out货管理

**功能说明：**
- 记录商品销售出库
- 库存不足自动提醒
- 自动更新库存
- 生成出货记录

**操作流程：**
1. 点击"新增出货"
2. 选择商品并输入数量（系统会验证库存）
3. 可添加多个商品到出货清单
4. 填写操作人和备注
5. 确认出货后自动扣减库存

**应用场景：**
- 销售给客户
- 活动物料出库
- 报损出库

### 📅 活动管理

**功能说明：**
- 管理各类业务活动
- 支持4种活动类型：促销活动、工作坊、展会、其他
- 物料分配管理
- 预算和利润核算

**创建活动：**
1. 点击"创建活动"
2. 填写活动基本信息（名称、类型、时间、地点）
3. 设置预算、实际成本、收入
4. 添加活动所需物料
5. 保存活动

**活动状态：**
- **筹备中**：活动正在准备阶段
- **进行中**：活动正在进行
- **已完成**：活动已结束
- **已取消**：活动取消

**财务核算：**
- 利润 = 收入 - 实际成本
- 系统会自动计算并以绿色（盈利）或红色（亏损）显示

### 💰 财务管理

**功能说明：**
- 记录收入和支出
- 按月统计财务数据
- 多种分类支持
- 实时利润计算

**支持的分类：**
- 销售收入
- 采购支出
- 活动相关
- 工资
- 租金
- 水电费
- 其他

**操作方法：**
1. 点击"添加记录"
2. 选择类型（收入/支出）和分类
3. 输入金额、日期、操作人
4. 填写详细描述
5. 保存记录

**财务报表：**
- 按月筛选查看
- 本月收入、支出、利润统计
- 支持按类型筛选

### 👥 客户管理

**功能说明：**
- 管理客户基础信息
- 记录联系方式
- 添加备注说明

**建议记录：**
- 客户全称
- 主要联系人
- 联系电话（必填）
- 邮箱、地址
- 特殊要求或备注

### 🏢 供应商管理

**功能说明：**
- 管理供应商信息
- 与商品关联
- 方便采购时选择

**使用技巧：**
- 在商品管理中可以直接填写供应商名称
- 建议详细记录供应商联系方式
- 在备注中记录合作条款、账期等信息

## 数据备份与恢复

### 导出数据（备份）

1. 点击顶部导航栏的"导出数据"按钮
2. 系统会下载一个JSON文件（文件名包含日期）
3. 建议定期备份，文件较小可存储在云盘

### 导入数据（恢复）

1. 点击顶部导航栏的"导入数据"按钮
2. 选择之前导出的JSON备份文件
3. 确认导入后页面会自动刷新
4. **注意**：导入会覆盖现有数据

**备份建议：**
- 每周至少备份一次
- 重要操作前先备份
- 保留多个历史备份
- 备份文件可用任何文本编辑器查看

## 使用技巧

### 1. 库存管理最佳实践

- **设置合理的预警值**：避免库存积压或缺货
- **定期盘点**：核对实际库存与系统记录
- **及时更新**：所有进出货都要及时登记
- **查看记录**：通过库存记录追溯问题

### 2. 活动管理技巧

- **提前规划**：创建活动时就分配好物料
- **实时更新**：活动进行中及时更新成本和收入
- **物料管理**：通过活动模块分配物料，方便成本核算
- **总结分析**：活动结束后查看利润，总结经验

### 3. 财务记录建议

- **及时记录**：每笔收支都要当天记录
- **分类清晰**：选择正确的分类便于后期分析
- **描述详细**：在描述中记录关键信息
- **定期查看**：每月查看财务报表，了解经营状况

### 4. 多人使用建议

- **约定规范**：统一商品命名、分类规则
- **操作人**：每次操作都填写操作人便于追溯
- **定期备份**：每天结束前由专人负责备份
- **权限约定**：明确谁可以删除数据

## 常见问题

**Q: 库存数字不对怎么办？**
A: 可以通过"库存管理"查看历史记录，找出问题。如需调整，可以通过进货或出货功能调整库存。

**Q: 误删了数据怎么恢复？**
A: 如果有备份文件，可以通过"导入数据"恢复。建议养成定期备份的习惯。

**Q: 如何查看某个商品的所有出入库记录？**
A: 进入"库存管理"，点击商品后面的"查看记录"，或在历史记录页面选择该商品。

**Q: 能否在手机上使用？**
A: 系统已适配移动端，可在手机浏览器中访问使用。

**Q: 数据存储在哪里？**
A: 数据存储在浏览器的IndexedDB中，只在本地存储，不会上传到服务器。

**Q: 多人如何同时使用？**
A: 当前版本是单机版，建议：
- 在一台电脑上使用，定期备份数据
- 或者通过远程桌面多人访问同一台电脑
- 如需真正的多人协同，需要部署服务器版本
